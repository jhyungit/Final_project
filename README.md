# 🍽 콘텐츠 기반 메뉴 추천 시스템

외식 구독 플랫폼 **달리셔스**의 고객·주문 데이터를 기반으로 개인화된 메뉴/메이커스를 추천하는 **콘텐츠 기반 추천 시스템(Content-Based Recommendation)** 입니다.

> T-academy(ASAC) 빅데이터 분석가 과정 **기업 연계 프로젝트** · 회사가 직접 제공한 실데이터 활용 · **우수상 (6팀 중 1위)**

---

## 📁 이 레포의 범위

이 공개 레포는 제가 담당한 **Content-Based 추천 모델 구현부**입니다.
전체 전처리 파이프라인, CF·ML·DL을 결합한 하이브리드 모델, 원본 데이터는 **기업 연계 프로젝트 특성상 팀 비공개(private) 레포**에 있습니다.

---

## 🎯 문제 정의

달리셔스 운영팀은 고객별 추천 메뉴를 **수동으로 선정**하고 있었고, 한 건당 약 30분이 소요되는 비효율 구조였습니다. 이를 **데이터 기반 자동 추천**으로 대체하는 것이 목표였습니다.

---

## 🗂 데이터

회사가 직접 제공한 고객·주문·메뉴 데이터(약 5만 건)를 사용했습니다. *(크롤링/외부 수집 아님)*

| 데이터 | 내용 |
|---|---|
| 주문 이력 | `Created` · `UserId` · `GroupId` · `FoodId` · `MakersId` |
| 음식 태그 | 음식별 태그 벡터 (콘텐츠 피처) |
| 메이커스/음식 매핑 | 서비스 가능 메이커스·음식 집합 |

---

## ⚙️ 구현 방법

### 1. 콘텐츠 피처 설계
- 음식 태그를 **메이커스 단위 태그 프로필**로 집계한 뒤 **L2 정규화(Normalizer)**
- 정규화된 프로필로 **메이커스–메이커스 / 음식–음식 코사인 유사도 행렬** 생성

```python
# 음식 태그를 메이커스 단위로 합산 → L2 정규화 → 코사인 유사도
maker_profile = makerstag.groupby('MakersId').sum()
maker_profile.iloc[:, :] = Normalizer().fit_transform(maker_profile)
makers_sim = cosine_similarity(maker_profile, maker_profile)
```

### 2. 최근 이용 기반 추천
- 각 유저/그룹이 **최근 이용한 N개(=5)의 메이커스·음식**을 프로필로 삼아, 유사도가 높은 항목을 추천
- 추천 축 3가지: **그룹 → 메이커스**, **유저 → 음식**, **유저 → 메이커스**

### 3. 콜드 스타트 대응
- 콘텐츠 기반이라 **주문 기록이 단 1건만 있어도 추천이 가능** — 상호작용 이력이 필요한 협업 필터링(CF)과 달리 신규·소량 이용 고객에게도 대응

## 📊 결과 및 성과

- 운영팀 수동 메뉴 선정 **30분 → 3분 (약 90% 단축)**
- 잠재고객 **2만 명 규모 고객사(SPARKPLUS)** 와 계약 성사에 기여
